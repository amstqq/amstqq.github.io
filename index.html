Visualizations

Shows and Nonprofits/Activism had the shortest average time to trending but also had very small samples. Travel & Events, Howto & Style, and Pets & Animals had the fastest time to trending among larger sample size categories. On the other hand, Autos and Film & Animation took the longest amount of days to trend on average.

Music and Gaming stayed trending for the longest number of days with large samples, whereas Sports and News/Politics trended for the shortest number of days.

This graph can be used to examine trend over time. Views for music category (orange line) started low in december but quickly took over as the most viewed category approaching may. Music videos are also more popular in the middle of a year than at the end of a year.

Visualization above broken down by category. This graph shows events in a given industry can influence views.

Film/Animation videos gained more views approaching February as Oscar was held in the same month. Another rise in views happened in May when major movies such as infinity war and deadpool 2 are released.

Weather API and JSON Traversals

Close Project

Skills: Web API, Piercing Data in JSON Format, Python Requests, Visualization Dashboard, CityPy

This project uses OpenWeatherMap API, Python requests, and JSON traversals to obtain weather data about 500+ random cities across the globe and create a representative model of weather trend. A visualization dashboard website is built with bootstrap using data collected from OpenWeatherMapAPI. The website contains all charts and graphs, explanations for weather trends, and data used to create the visualizations. Check out Github repository of this project for more information.

API Calls and Visualization Dashboard

All weather data is obtained from OpenWeatherMap API. A list of more than 500 Random cities are selected based on latitude and longitude and a weather check is performed on each of the cities using a series of successive API calls. The visualization dashboard can be found here.

Honolulu Climate SQL Database and Flask API

Close Project

Skills: SQLAlchemy, SQL Database, ORM Queries, Flask API

This project uses SQLAlchemy ORM queries in Python to explore and create visualizations of Honolulu climate database containing temperature and precipitation data. SQLAlchemy is a SQL toolkit in Python that allows control of SQL database in Python scripts. Then, a Flask API is designed to assess the climate database through SQLAlchemy and return the results in json format.

Climate Data Analysis

The graph shows precipitation data from two years ago to today. The data is obtained by reflecting SQL database, linking it to Python, and query the desired data.

Query temperature observation data and plot the results as a histogram.

The average temperature of a certain range of data is plotted. A function is built in Python to accept start and end dates as parameters and return minimum, average, and maximum temperature for that range of dates.

This graph is obtained by plotting the daily minimum, average, and maximum tempeature for a given range of dates.

Flask API

This page is the default route of Flask API. It gives a complete list of available routes and data that can be assessed using this API.

Minimum, average, maximum tempeature in a specific date range is returned by inputting a 'start_date' and 'end_date' in the route /api/v1.0/start_date/end_date. For example, the temperature data in this photo is from the route /api/v1.0/2016-08-23/2017-04-23.

Precipitation data in the last two years is returned in the route /api/v1.0/precipitation.

A list of all tempeature recording stations is retured in the route /api/v1.0/stations.
